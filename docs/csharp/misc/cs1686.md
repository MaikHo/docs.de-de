# <a name="compiler-error-cs1686"></a><span data-ttu-id="b8a07-102">Compilerfehler CS1686</span><span class="sxs-lookup"><span data-stu-id="b8a07-102">Compiler Error CS1686</span></span>

<span data-ttu-id="b8a07-103">Die Adressen von "Variable" (lokal) oder der entsprechenden Member können nicht übernommen und in einer anonymen Methode oder einem lambda-Ausdruck verwendet werden.</span><span class="sxs-lookup"><span data-stu-id="b8a07-103">Local 'variable' or its members cannot have their address taken and be used inside an anonymous method or lambda expression</span></span>  
  
 <span data-ttu-id="b8a07-104">Dieser Fehler wird generiert, wenn Sie eine Variable verwenden und versuchen, deren Adresse zu übernehmen und eine dieser Aktionen in einer anonymen Methode erfolgt.</span><span class="sxs-lookup"><span data-stu-id="b8a07-104">This error is generated when you use a variable, and attempt to take its address, and one of these actions is done inside an anonymous method.</span></span>  
  
## <a name="example"></a><span data-ttu-id="b8a07-105">Beispiel</span><span class="sxs-lookup"><span data-stu-id="b8a07-105">Example</span></span>  

 <span data-ttu-id="b8a07-106">Im folgenden Beispiel wird CS1686 generiert:</span><span class="sxs-lookup"><span data-stu-id="b8a07-106">The following sample generates CS1686.</span></span>  
  
```csharp  
// CS1686.cs  
// compile with: /unsafe /target:library  
class MyClass  
{  
   public unsafe delegate int * MyDelegate();  
  
   public unsafe int * Test()  
   {  
      int j = 0;  
      MyDelegate d = delegate { return &j; };   // CS1686  
      return &j;   // OK  
   }  
}  
```
