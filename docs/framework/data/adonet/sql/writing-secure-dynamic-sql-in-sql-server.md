<span data-ttu-id="bcbd8-103">Bei der Einschleusung von SQL-Befehlen gibt ein böswilliger Benutzer anstelle gültiger Eingaben Transact-SQL-Anweisungen ein.</span><span class="sxs-lookup"><span data-stu-id="bcbd8-103">SQL Injection is the process by which a malicious user enters Transact-SQL statements instead of valid input.</span></span> <span data-ttu-id="bcbd8-104">Wenn die Eingabe ohne Überprüfung direkt an den Server weitergegeben wird und die Anwendung den eingeschleusten Code versehentlich ausführt, kann der Angriff Daten beschädigen oder zerstören.</span><span class="sxs-lookup"><span data-stu-id="bcbd8-104">If the input is passed directly to the server without being validated and if the application inadvertently executes the injected code, the attack has the potential to damage or destroy data.</span></span>  
  
 <span data-ttu-id="bcbd8-105">Sie sollten jede Prozedur, die SQL-Anweisungen erstellt, nach Injection-Anfälligkeiten überprüfen, denn SQL Server führt alle empfangenen gültigen Abfragen aus.</span><span class="sxs-lookup"><span data-stu-id="bcbd8-105">Any procedure that constructs SQL statements should be reviewed for injection vulnerabilities because SQL Server will execute all syntactically valid queries that it receives.</span></span> <span data-ttu-id="bcbd8-106">Auch parametrisierte Daten können von einem technisch versierten Angreifer manipuliert werden.</span><span class="sxs-lookup"><span data-stu-id="bcbd8-106">Even parameterized data can be manipulated by a skilled and determined attacker.</span></span> <span data-ttu-id="bcbd8-107">Wenn Sie dynamisches SQL verwenden, stellen Sie sicher, dass Sie Ihre Befehle parametrisieren und Parameterwerte niemals direkt in die Abfragezeichenfolge einschließen.</span><span class="sxs-lookup"><span data-stu-id="bcbd8-107">If you use dynamic SQL, be sure to parameterize your commands, and never include parameter values directly into the query string.</span></span>

 <span data-ttu-id="bcbd8-109">Die Funktionalität des Injection-Prozesses besteht in der vorzeitigen Beendigung einer Textzeichenfolge und dem Anhängen eines neuen Befehls.</span><span class="sxs-lookup"><span data-stu-id="bcbd8-109">The injection process works by prematurely terminating a text string and appending a new command.</span></span> <span data-ttu-id="bcbd8-110">Da vor der Ausführung des eingefügten Befehls möglicherweise weitere Zeichenfolgen daran angehängt wurden, beendet der Missetäter die eingefügte Zeichenfolge durch eine Kommentarmarkierung "--".</span><span class="sxs-lookup"><span data-stu-id="bcbd8-110">Because the inserted command may have additional strings appended to it before it is executed, the malefactor terminates the injected string with a comment mark "--".</span></span> <span data-ttu-id="bcbd8-111">Nachfolgender Text wird zur Ausführungszeit ignoriert.</span><span class="sxs-lookup"><span data-stu-id="bcbd8-111">Subsequent text is ignored at execution time.</span></span> <span data-ttu-id="bcbd8-112">Mehrere Befehle können mit einem Semikolon (;) als Trennzeichen eingefügt werden.</span><span class="sxs-lookup"><span data-stu-id="bcbd8-112">Multiple commands can be inserted using a semicolon (;) delimiter.</span></span>  
  
 <span data-ttu-id="bcbd8-113">Solange der eingefügte SQL-Code syntaktisch richtig ist, kann eine Manipulation nicht programmgesteuert erkannt werden.</span><span class="sxs-lookup"><span data-stu-id="bcbd8-113">As long as injected SQL code is syntactically correct, tampering cannot be detected programmatically.</span></span> <span data-ttu-id="bcbd8-114">Aus diesem Grund müssen Sie die Gültigkeit aller Benutzereingaben überprüfen und sorgfältig den Code überprüfen, der in dem von Ihnen verwendeten Server die erstellten SQL-Befehle ausführt.</span><span class="sxs-lookup"><span data-stu-id="bcbd8-114">Therefore, you must validate all user input and carefully review code that executes constructed SQL commands in the server that you are using.</span></span> <span data-ttu-id="bcbd8-115">Verketten Sie niemals Benutzereingaben, die nicht überprüft wurden.</span><span class="sxs-lookup"><span data-stu-id="bcbd8-115">Never concatenate user input that is not validated.</span></span> <span data-ttu-id="bcbd8-116">Die Zeichenfolgenverkettung ist der primäre Eingangspunkt für Script-Injection.</span><span class="sxs-lookup"><span data-stu-id="bcbd8-116">String concatenation is the primary point of entry for script injection.</span></span>  
  
 <span data-ttu-id="bcbd8-117">Hier einige nützliche Leitlinien:</span><span class="sxs-lookup"><span data-stu-id="bcbd8-117">Here are some helpful guidelines:</span></span>

- <span data-ttu-id="bcbd8-124">Testen Sie den Inhalt der Zeichenfolgenvariablen, und akzeptieren Sie nur erwartete Werte.</span><span class="sxs-lookup"><span data-stu-id="bcbd8-124">Test the content of string variables and accept only expected values.</span></span> <span data-ttu-id="bcbd8-125">Lehnen Sie Einträge mit Binärdaten, Escapesequenzen und Befehlszeichen ab.</span><span class="sxs-lookup"><span data-stu-id="bcbd8-125">Reject entries that contain binary data, escape sequences, and comment characters.</span></span>

- <span data-ttu-id="bcbd8-128">Folgende Zeichenfolge dürfen in Feldern, aus denen Dateiname erstellt werden können, nicht zugelassen werden: AUX, CLOCK$, COM1 bis COM8, CON, CONFIG$, LPT1 bis LPT8, NUL und PRN.</span><span class="sxs-lookup"><span data-stu-id="bcbd8-128">Do not accept the following strings in fields from which file names can be constructed: AUX, CLOCK$, COM1 through COM8, CON, CONFIG$, LPT1 through LPT8, NUL, and PRN.</span></span>
